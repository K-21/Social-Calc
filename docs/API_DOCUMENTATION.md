# Spreadsheet API Documentation

This document describes the JSON-based API endpoints used for spreadsheet import and export interoperability between the C# backend and the JavaScript (SheetJS) frontend.

## 1. Import Endpoint
**POST `/api/spreadsheet/import`**

This endpoint accepts a parsed workbook payload from the frontend (generated by reading an Excel/CSV file using SheetJS) and creates a new spreadsheet in the database, mapping the structure to SocialCalc's internal `savestr` format.

### Request Body (JSON)
```json
{
  "fileName": "My Imported Sheet",
  "targetSheetId": null, // (Optional) If provided, overwrites an existing sheet instead of creating a new one
  "sheets": [
    {
      "name": "Sheet1",
      "cells": [
        {
          "r": 0,       // Row index (0-based)
          "c": 0,       // Column index (0-based)
          "v": "Hello", // Raw value
          "t": "s",     // Cell type ('s' for string, 'n' for number, 'b' for boolean)
          "f": null     // (Optional) Formula string, e.g., "SUM(A1:A2)"
        }
      ],
      "merges": [
        {
          "s": { "r": 0, "c": 0 }, // Start row/col
          "e": { "r": 0, "c": 2 }  // End row/col
        }
      ],
      "cols": [
        { "index": 0, "size": 120 } // Column width in px
      ],
      "rows": [
        { "index": 0, "size": 20 }  // Row height in pt
      ]
    }
  ]
}
```

### Response (JSON)
```json
{
  "success": true,
  "id": 123,
  "message": "Imported successfully"
}
```

## 2. Export Endpoint
**GET `/api/spreadsheet/export/{id}`**

This endpoint retrieves a saved spreadsheet from the database, parses its native SocialCalc `savestr` format, and converts it into a standardized JSON workbook payload that the frontend can pass to SheetJS for downloading as Excel/CSV.

### Request Headers
- `RequestVerificationToken`: Sent by the browser, but not validated on the GET endpoint.

### Response (JSON)
The response shape exactly mirrors the `sheets` payload structure expected by the Import endpoint.

```json
{
  "fileName": "My Spreadsheet",
  "sheets": [
    {
      "name": "Sheet1",
      "cells": [
        { "r": 0, "c": 0, "v": "Hello", "t": "s", "f": null }
      ],
      "merges": [],
      "cols": [],
      "rows": []
    }
  ]
}
```

## Known Limitations
* **Styling Support:** Background colors, custom fonts, borders, and complex formatting are **not currently mapped** between Excel and SocialCalc. The API strictly focuses on transferring raw values, types, and formulas.
* **Multi-Sheet Workbooks:** SocialCalc supports multi-sheet workbooks in theory, but the UI currently restricts the user to a single active sheet ("Sheet1"). Importing an Excel file with multiple sheets will append them, but they may not render correctly in the single-sheet UI mode.
* **Formula Conversion:** Basic standard Excel formulas (e.g., `=SUM(A1:B1)`) are mapped exactly. Highly complex proprietary Excel formulas will be preserved as strings but may not compute correctly inside the SocialCalc engine.
